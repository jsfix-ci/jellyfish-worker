tick

  • tick(context: LogContext, session: string, options: { currentDate: number | Date }): Promise<void>
    • Defined in lib/index.ts:986
    summary

    Execute a worker tick

    function
    description

    A tick is necessary to dispatch time-triggered actions and potentially any other logic that depends on the concept of time.

    Applications should "tick" on a certain interval. Shorter intervals increase the accuracy of time-related actions, but introduces more overhead.

    The tick operation may enqueue new actions but will not execute them right away.

    example

    const worker = new Worker({ ... }) const session = '4a962ad9-20b5-4dd8-a707-bf819593cc84'

    await worker.tick({ ... }, session, { currentDate: new Date() })

    Parameters

    • context: LogContext

      execution context

    • session: string

      session id

    • options: { currentDate: number | Date }

      options

      • currentDate: number | Date

        current date

    Returns Promise<void>
